### Phase 4：人工确认更新

**4.1 展示更新摘要**

```

 ARCHIVE — 知识更新确认
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**新业务规则 ({N} 条)：**
  1. {规则描述}（来源: {文件}）
  2. {规则描述}（来源: {文件}）

**新代码模板 ({N} 个)：**
  1. {模板名} — {适用场景}（样本数: {N}）
  2. {模板名} — {适用场景}（样本数: {N}）

**AGENTS.md 更新 ({N} 项)：**
  1. {更新内容描述}
  2. {更新内容描述}

**domain.md 更新 ({N} 项)：**
  1. {更新内容描述}
  2. {更新内容描述}

**模板 Index 更新：**
  + {新增的模板条目}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**4.2 逐项确认**

使用 AskUserQuestion 逐类确认：

```
AskUserQuestion:
  question: |
    请确认以下知识更新：

    **新业务规则 ({N} 条)**：
    {逐条列出}

    是否写入知识库？
  options:
    - 全部写入
    - 部分写入（请说明保留哪些）
    - 跳过，不写入
```

同理对新代码模板和 AGENTS.md 更新分别确认。

**4.3 执行更新**

根据确认结果执行写入：

**写入新业务规则：**
- 追加到 `.vibe/domain.md` 的对应章节（如校验规则、状态流转规则、约束规则等）
- 新术语追加到 domain.md 的领域术语表
- 格式遵循 domain.md 现有结构

**写入新代码模板：**
- 新模板写入 `.vibe/templates/` 目录（每个模板独立文件，命名规范：`flow-{name}.md` 或 `capability-{name}.md`）
- 模板格式遵循 Day0 的 Flow Bundle Template 或 Capability Template 格式（含 YAML frontmatter）
- 写入完成后，**自动调用 `/templates-index`** 重新生成 `.vibe/index.md`

**更新 AGENTS.md：**
- 在现有内容基础上补充
- 不覆盖已有规则
- 变更前展示 diff 摘要
